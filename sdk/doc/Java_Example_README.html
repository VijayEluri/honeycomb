@HoneycombFullProductName@ Java Examples

Included with the @HoneycombProductName@ SDK are several command line example applications that demonstrate the Java API. The example applications are located in the SDK under java/examples. These example applications are designed to be compiled with Java 1.5 or higher and come complete with a build script. The following Java examples are included:

StoreFile
Allows you to specify a file from the command line to store on a @HoneycombProductName@ as data. Also allows you to specify metadata to include with the file. view source code

AddMetadata
Allows you to specify name value pairs from the command line to store on a @HoneycombProductName@ as a metadata record. view source code

RetrieveData
Allows you to retrieve data from the @HoneycombProductName@ associated with the supplied Object Identifier. The data is saved to a file specified on the command line. view source code

RetrieveMetadata
Allows you to retrieve a metadata record from the @HoneycombProductName@ associated with the supplied Object Identifier. The metadata record is printed to standard output. view source code

DeleteRecord
Deletes a Record from the @HoneycombProductName@ for an Object Identifier supplied on the command line. If successful returns the OID of the deleted object to standard output. view source code

Query
Query a @HoneycombProductName@ for specified Metadata. The query string is provided on the command line. Query can print out the results as a list of name/value pairs or as a list of OIDs. Query allows you to specify the maximum number of results returned. view source code

RetrieveSchema
Retrieves the metadata schema from a @HoneycombProductName@ printing it to standard output. view source code

How to Build the Example Applications

The example applications are in the java/examples folder in the SDK. To build the example applications execute the master_build.sh script for Linux/Solaris or the master_build.bat script for Windows. This will build the examples and put them in a jar archive located in java/lib/ (note that this has already been done for you, so it isn't necessary unless you want to change the programs and recompile). The compile warning, "Some input files use unchecked or unsafe operations" can be ignored - it is a Java 1.5 artifact of using a HashMap to store a variety of object types.

How to run the Example Applications

Once you have built the example applications go to the java/examples/build folder and execute the jar files:

java -jar <Jar File Name> <arguments>

The examples depend on Getopt.jar and honeycomb-client.jar. These files must be in the same directory as the example jar files in order for them to run.

About the Example Application Source Code

The example applications are simple and all follow the same basic pattern. First, Commandline.parse is called to parse the argument list. Next, the appropriate method in in the NameValueObjectArchive class is called to communicate with the @HoneycombProductName@. Finally, output is delivered back to either standard output, a file, or both. Please refer to the comments in the sample code for further details.
